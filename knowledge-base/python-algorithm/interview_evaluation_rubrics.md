# Interview Evaluation Rubrics (Python算法工程师 / 大模型架构师)

本文档旨在统一规范基于 **Python基建、机器学习基础理论、大模型原理及应用实操** 等各个切面的考核深度与统一标尺。

## 一、 考核大盘拆分权重
整套定级打分采用以下 5个矩阵切块作为加权依据：
1. **Python 工程护城河 (15%)**
2. **机器学习与理论算法 (20%)**
3. **LLM与深度学习基建层 (25%)**
4. **大语言工程提效与RAG架构落地 (25%)**
5. **软素养、抗压与技术判断视野 (15%)**

---

## 二、 详细评分标准矩阵

### 2.1 Python底层与工程健壮度 (15%)
作为AI的地基，主要考察是否能应对高负荷部署时的数据流及防内存溢出：
- **专家 (4-5分)**：熟练分析 CPython 中 GIL 对多核并行的限制；针对OOM能提出分代垃圾回收机制、通过 `weakref` 弱引用消除循环引用；深入理解 Numpy 中 strides（步幅）实现零拷贝的原理。
- **合格 (2.5-3.5分)**：可用装饰器构建无感耗时埋点闭包栈；熟悉生成器应对GB级数据集不爆内存的运用；明白深浅拷贝机制导致的联动引用Bug。
- **不合格 (<2分)**：不知道迭代器与生成器的区别，认为Python开启内置的多线程就能处理海量并行的密集运算，基础数据结构时空复杂度乱判。

### 2.2 基础机器学习与演变能力 (20%)
算法基本功排查，考察应对小算力和结构化表单时候的数据挖掘与模型搭建：
- **专家 (4-5分)**：能深度分析 Random Forest（Bagging）和 XGBoost（Boosting）的数理差异（偏差-方差权衡）；能推导交叉熵损失函数如何避免 Sigmoid 梯度消失问题；深入理解 Adam 优化器如何结合动量（一阶矩）与自适应学习率（二阶矩）。
- **合格 (2.5-3.5分)**：理解 L1 正则化产生稀疏解、L2 正则化平滑权重防过拟合的原理；能在样本不平衡场景中使用 Focal Loss 和 SMOTE 重采样，采用 AUC 等指标替代准确率进行评估。
- **不合格 (<2分)**：只懂`import sklearn`掉包跑分类；不能表述出交叉熵跟 MSE(均方误差) 的使用区别，无法讲清楚超参数之间的权衡。

### 2.3 LLM与深度构架体系 (25%)
要求深入理解模型底层原理，而非仅会调用 API：
- **专家 (4-5分)**：能推导 Transformer 中 Attention 通过缩放因子（根号dk）归一化方差以防止梯度消失；精通 Decoder-only 架构的自回归生成机制；能解释 RoPE（旋转位置编码）基于复平面的相对位置外推原理。
- **合格 (2.5-3.5分)**：讲得出残差连接解决梯度消失的过程；知道 Pre-Norm 和 Post-Norm 以及 RMSNorm 在层迭代深度的防止发散保护；
- **不合格 (<2分)**：对于 QKV 是什么、怎么点积乘的一无所知；对词表切割（BPE / 分词）未登录词规避完全无概念。

### 2.4 大模型算力极限与部署工程 (25%)
考核模型微调的成本控制和工程部署能力：
- **专家 (4-5分)**：熟悉 DeepSpeed ZeRO 三阶段显存优化机制（优化器状态/梯度/参数分片）；深入理解 KV-Cache 机制及 MQA/GQA（多查询/分组查询注意力）的显存优化原理；掌握 vLLM 的 PagedAttention 分页调度技术。
- **合格 (2.5-3.5分)**：熟悉 LoRA（低秩适配）等 PEFT 参数高效微调原理并有实践经验；具备完整 RAG 系统调试经验（文档分块、Embedding 微调、重排序机制），能有效缓解大模型幻觉问题。
- **不合格 (<2分)**：只会在平台点训练。在碰到单张显存24G显卡不够部署的情况下，不能提出模型量化或任何减少显存支出的降维技术手段。

### 2.5 软素养、问题排查及沟通协作 (15%)
评估候选人处理训练异常和跨团队协作的能力：
- **专家 (4-5分)**：微调出现模型能力退化时，能回滚基座并分析灾难性遗忘（Catastrophic Forgetting）原因进行数据清洗和对齐修复；面对不合理需求时能提出知识蒸馏等替代方案平衡效果与资源。
- **合格 (2.5-3.5分)**：训练出现 Loss 为 NaN 时，能定位梯度爆炸或混合精度溢出问题。能够客观评估技术可行性，与产品团队有效沟通。
- **不合格 (<2分)**：将模型效果问题归咎于"玄学"而不进行深入分析。盲目接受不合理需求导致线上故障。
