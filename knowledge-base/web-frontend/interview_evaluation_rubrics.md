# 前端面试核心考点评分要点

本文档提取了前端篇的所有高频考点，作为面试评分标准参考，涵盖 HTML/CSS、JavaScript、计算机网络、Vue/React、Webpack工程化等核心领域。

## HTML / CSS

### 1. DOCTYPE与浏览器渲染模式
- **必须提到（基础分）**：DOCTYPE用于声明HTML文档类型，告诉浏览器用什么标准解析网页。
- **应该提到（中等分）**：如果缺失，浏览器会进入怪异模式（混杂模式）进行向后兼容展示。
- **高分项（加分）**：说明标准模式按W3C最高标准渲染，怪异模式则由于各家实现不同会导致页面布局错乱兼容性问题。
- **常见错误**：误以为这是引入外部库或执行某种代码的强制性标签。

### 2. HTML语义化
- **必须提到（基础分）**：语义化是指根据内容结构选择合适的标签（如header/footer/nav代替无意义的div）。
- **应该提到（中等分）**：增加代码可读性、对机器友好（利于屏幕阅读器）、极大提升SEO。
- **高分项（加分）**：在无CSS情况下，合理的语义页面依然能呈现清晰的大纲级别内容树结构。
- **常见错误**：仅仅把语义标签当做带了默认CSS边距的特殊占位符。

### 3. src与href的区别
- **必须提到（基础分）**：href是引用网络资源建立链接，src是下载资源并且替换掉当前元素。
- **应该提到（中等分）**：href并行下载且不阻塞文档解析(如link)，而src会暂停文档编译执(如script)。
- **高分项（加分）**：结合这点解释为何通常要把<script>标签放在页面底部防渲染阻塞。
- **常见错误**：把二者视为仅仅是不同英语单词拼写的通用请求。

### 4. 重绘(Repaint)与重排(Reflow)的区别及优化
- **必须提到（基础分）**：重排是元素尺寸/结构变化引起布局重新计算，重绘是外观(颜色/背景)改变但不影响布局。重排必定引发重绘。
- **应该提到（中等分）**：防抖和离线DOM操作（DocumentFragment）、修改class名集中更新可以减少重排。
- **高分项（加分）**：利用GPU硬件加速（CSS的transform和opacity不会触发重排重绘）。
- **常见错误**：将visibility: hidden(重绘)和display: none(重排)的作用混为一谈。

### 5. BFC (块级格式化上下文)
- **必须提到（基础分）**：BFC是一个独立的渲染区域，内部元素不影响外部。
- **应该提到（中等分）**：触发条件包括float不为none，position为absolute/fixed，overflow不为visible，display为flex/inline-block等。
- **高分项（加分）**：说明常用场景：解决两兄弟元素的margin重叠问题，以及父元素清除子元素浮动防塌陷。
- **常见错误**：只能死背触发条件，不知道它本质上就是隔离内部浮动与外部文档流的结界。

## JavaScript 核心原理

### 6. 闭包 (Closure)
- **必须提到（基础分）**：函数和其周围状态(词法作用域)的引用捆绑在一起形成闭包。能让函数访问其外部作用域。
- **应该提到（中等分）**：用途包含数据私有化、科里化函数、模块模式。缺点是会导致变量不会被垃圾回收，引起内存泄漏。
- **高分项（加分）**：结合JS执行上下文(Execution Context)和作用域链(Scope Chain)深度剖析。
- **常见错误**：认为闭包就是函数套函数，忽略了“保持外部变量引用”的核心要素。

### 7. 原型和原型链 (Prototype Chain)
- **必须提到（基础分）**：每个对象都有 `__proto__` 内部属性指向它的原型对象(除了null)，形成链式结构。
- **应该提到（中等分）**：当访问对象属性时，找不到则会顺着原型链层层往上找。所有函数的prototype都会被其实例的__proto__链接。
- **高分项（加分）**：画出Function和Object之间错综复杂的原型指向（`Function.__proto__ === Function.prototype`）。
- **常见错误**：混淆了对象的隐式原型(`__proto__`)与构造函数的显式原型(`prototype`)。

### 8. this指向规则
- **必须提到（基础分）**：this的指向由函数的“调用方式”决定，而不是定义位置（箭头函数除外）。
- **应该提到（中等分）**：默认绑定(全局/undefined)、隐式绑定(调用它的对象)、显式绑定(call/apply/bind)、new绑定(实例对象)。
- **高分项（加分）**：指出箭头函数中没有自己的this，其this由外层代码块的词法作用域决定且不可用bind更改。
- **常见错误**：认为在对象的方法里面写了普通函数，其内部的this依旧指向该对象（实际若未绑定则指向全局）。

### 9. 事件循环机制 (Event Loop)
- **必须提到（基础分）**：JS是单线程的，遇到异步任务交给宿主环境维护，回调完成后推入队列，主线程按序取用。
- **应该提到（中等分）**：区分宏任务（setTimeout, I/O）与微任务（Promise.then, MutationObserver）。执行逻辑是“取出宏任务 -> 执行微任务队列 -> 渲染”。
- **高分项（加分）**：Node.js环境下的Event Loop有六个阶段（Timers, I/O, Poll等），并说明 process.nextTick 优先级极高。
- **常见错误**：以为宏任务是包含了整个一次循环的代码，不理解清空微任务是附着在每个宏任务执行末尾的。

### 10. Promise 及异步编程方案发展
- **必须提到（基础分）**：Promise解决回调地狱，有三种状态：Pending, Fulfilled, Rejected。状态不可逆。
- **应该提到（中等分）**：区别Promise.all() (只要有一个失败即失败) 和 Promise.race() (最先出结果的为准)。
- **高分项（加分）**：梳理JS异步史：Callback -> Promise -> Generator(yield) -> async/await语法糖（被编译也是对应Promise状态流转）。
- **常见错误**：认为 Promise 本身（传入构造函数的执行器体）是异步的，其实构造执行器内代码是同步立刻执行，只有 then 才是异步微任务。

## 计算机网络 & 浏览器实现

### 11. TCP 三次握手和四次挥手
- **必须提到（基础分）**：明确指出建连需3次通讯，断连需4次通信。
- **应该提到（中等分）**：解释之所以三次是为了确保双方“发和收”能力正常；四次挥手是为了处理全双工状态下的半关闭(等待残留数据发完)。
- **高分项（加分）**：如果两次握手会有什么灾难：客户端丢失的SYN如果延迟到达，服务端会瞎开资源导致浪费等待死锁。
- **常见错误**：强行生搬硬背报文标识位却说不通为什么要进行确认。

### 12. 跨域问题及其解决方案
- **必须提到（基础分）**：受浏览器同源策略（协议、域名、端口相同）限制。常见方案有CORS。
- **应该提到（中等分）**：CORS需要后端配置 Access-Control-Allow-Origin。JSONP利用script只支持GET；本地开发常用Node反向代理解决。
- **高分项（加分）**：CORS发起的Options预检请求（Preflight）发生的条件（复杂请求导致），并说明Cookie跨域时需携带 credentials。
- **常见错误**：以为JSONP是AJAX的一种变体，或以为在服务端与服务端之间存在跨域限制。

### 13. 从输入URL到页面展示发生了什么
- **必须提到（基础分）**：DNS解析 -> TCP建连 -> HTTP请求 -> 服务端响应 -> 浏览器解析渲染。
- **应该提到（中等分）**：浏览器解析的内部流程：解析HTML为DOM树，CSS为CSSOM树，合并为Render Tree，接着进行Layout布局和Paint绘制。
- **高分项（加分）**：深入探讨浏览器进程模型，讲清楚网络进程拿数据IPC给渲染进程的主线程，并将Composite合成层转给GPU进程加速。
- **常见错误**：漏掉绘制和合成的知识点，仅仅停留在网络层面。

## 框架(Vue/React)与工程化基础

### 14. 虚拟DOM (Virtual DOM) 与 Diff算法
- **必须提到（基础分）**：虚拟DOM是用JS对象描述真实的DOM结构，减少直接操作真实DOM的昂贵成本。
- **应该提到（中等分）**：每次状态变化生成新VNode，通过Diff找出差异再打补丁(Patch)到真实DOM。它提供了跨端能力(如RN/跨平台)。
- **高分项（加分）**：Diff算法同层比较机制；由于key的作用避免不必要的节点重建；React的Fiber中可中断比对机制。
- **常见错误**：认为Virtual DOM一定比原生DOM操作快（在极简DOM变动下原生更优）。

### 15. Vue双向绑定核心原理
- **必须提到（基础分）**：Vue采用数据劫持 + 发布订阅模式实现响应式。
- **应该提到（中等分）**：Vue2使用 Object.defineProperty 劫持getter/setter，Vue3因为性能及全方位代理需求升级成了ES6 Proxy。
- **高分项（加分）**：解释Dep（依赖收集器）和Watcher（侦听者）的运作关联，以及Vue2对数组Hack的7个重写方法原理。
- **常见错误**：只答了defineProperty，完全不知道还需要依赖收集(被读取时)和派发更新(被赋值时)。

### 16. Webpack的 Loader 和 Plugin
- **必须提到（基础分）**：Webpack默认只认识JS/JSON。Loader用于解析转换别的文件；Plugin用于扩展构建能力。
- **应该提到（中等分）**：常见Loader(babel-loader, css-loader, sass-loader)是从右向左执行的。常见Plugin包括HtmlWebpackPlugin, CleanWebpackPlugin。
- **高分项（加分）**：Loader其实是个函数，Plugin本质是个通过apply方法接入编译器各个Hook(钩子)生命周期的类。
- **常见错误**：把压缩代码的任务当做Loader的工作。
